Лабораторна робота №7: Комплексний захист особистого проекту
Опис
Цей проект розроблено в рамках дисципліни «Захист інформації» для реалізації комплексної системи захисту даних. Програма демонструє поєднання двох методів захисту: криптографічного (шифрування) та стеганографічного (приховування факту існування даних), що забезпечує дворівневу безпеку.

Функціонал
Програма main.py реалізує автоматизований конвеєр захисту даних з наступними функціями:

Генерація ключів: Створення криптографічного ключа на основі персональних даних (Email) за допомогою алгоритму хешування SHA-256.

Шифрування (Етап 1): Захист конфіденційності вмісту файлу методом XOR-шифрування (симетричний алгоритм).

Стеганографія (Етап 2): Приховування зашифрованих даних у контейнері (імітація зображення) методом LSB (Least Significant Bit).

Контроль цілісності: Використання контрольної суми CRC32 для виявлення пошкоджень або спроб несанкціонованої модифікації контейнера.

Аналітика: Збір метрик ефективності (час обробки, розмір файлів).

Системні Вимоги
Python 3.10+

Стандартні бібліотеки Python: os, time, hashlib, random, zlib.

Запуск Програми
Для запуску демонстрації виконайте команду в терміналі:

Bash

python main.py
Сценарій Демонстрації
Скрипт працює в автоматичному режимі та демонструє 6 етапів роботи системи:

1. Вхідні дані
Генерується тестовий документ ("Confidential data...") та імітується зображення-контейнер. Ключ генерується на основі персональних даних.

2. Захист (Шифрування + Стеганографія)
Дані шифруються, перетворюючись на нечитабельний байт-код.

Шифротекст вбудовується в контейнер.

Результат: Файл protected_image.bin, який візуально не відрізняється від оригіналу (розмір не змінюється).

3. Відновлення
Система вилучає дані з контейнера та дешифрує їх за допомогою правильного ключа.

4. Перевірка цілісності
Програма звіряє хеш вилучених даних з оригіналом.

Результат: ✓ Документ відновлено успішно!

5. Тест на підробку
Емуляція атаки "Man-in-the-Middle": змінюється один байт у захищеному контейнері.

Результат: Система виявляє втручання завдяки невідповідності CRC32 (✓ Підробку виявлено: дані пошкоджені).

6. Спроба злому (Неправильний ключ)
Спроба дешифрування даних за допомогою неправильного ключа.

Результат: Отримуються нечитабельні символи ("сміття"), конфіденційність збережено.

Технічні деталі реалізації
Алгоритм шифрування: XOR (побітове виключне АБО) з циклюванням ключа. Забезпечує швидкість та простоту реалізації.

Стеганографія: Метод LSB (вбудовування у молодші біти). У даній реалізації для наочності використовується запис у байтовий масив контейнера.

Захист від помилок: Використано бібліотеку zlib для розрахунку CRC32, що дозволяє гарантувати цілісність даних при їх вилученні.

Висновки
Розроблена система (Комбінація А) довела свою ефективність. Поєднання шифрування та стеганографії створює надійний захист: навіть якщо зловмисник виявить факт передачі даних (зламає стеганографію), він не зможе прочитати їх без ключа шифрування.
