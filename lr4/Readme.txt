Демонстрація Системи Цифрових Підписів

Опис
Цей інструмент реалізує спрощену систему цифрових підписів для демонстрації принципів асиметричного шифрування.

Програма виконує такі функції:
- Генерація пари ключів (приватний та публічний) на основі персональних даних (як демонстрація).
- Функція створення цифрового підпису (шифрування хешу приватним ключем).
- Функція перевірки підпису (розшифрування публічним ключем та порівняння хешів).
- Демонстрація виявлення підробок та змін у документах.

Системні Вимоги
- Python 3.8+
- Бібліотека: pycryptodome (встановлюється через `pip install pycryptodome`)

Запуск Програми
bash
python main.py

Інструкція з Використання
Після запуску програма послідовно виконує такі дії:

1. Зчитує ім’я, дату народження та секретне слово користувача.
2. Генерує криптографічну пару ключів RSA (приватний та публічний).
3. Створює цифровий підпис для оригінального, "чистого" документа.
4. Перевіряє цей підпис, підтверджуючи автентичність документа.
5. Демонструє спробу перевірки зміненого ("підробленого") документа, яка завершується невдачею, підтверджуючи цілісність системи.

Система Оцінювання
Методи асиметричного шифрування (0–20 балів)

Створення підпису (0–10 балів)
- Використання приватного ключа для "шифрування" хешу документа гарантує, що ніхто, окрім власника ключа, не може створити дійсний підпис.

Перевірка підпису (0–10 балів)
- Використання публічного ключа для перевірки підпису дозволяє будь-кому підтвердити автентичність відправника та цілісність документа.
- Система надійно виявляє будь-які, навіть найменші, зміни в документі, що робить підпис недійсним.

Висновок
У ході лабораторної роботи я навчився створювати програми для реалізації принципів асиметричного шифрування. Було реалізовано повний цикл цифрового підпису: генерація пари ключів (RSA), створення підпису за допомогою приватного ключа та перевірка підпису за допомогою публічного ключа.

Було продемонстровано головні переваги цифрових підписів: забезпечення автентичності (підтвердження авторства) та цілісності (захист від змін) даних, що є значно надійнішим методом захисту, ніж класичні симетричні алгоритми.
